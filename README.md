# handy-roast-logger
https://210on.github.io/handy-roast-logger/

スマートフォンでの操作に特化した、音声入力対応のコーヒー焙煎ロガーです。
焙煎中の「手が離せない」「画面を注視できない」という課題を解決するため、極限までノイズを削ぎ落としたUIと、1つのボタンで完結するシームレスな体験（UX）を提供します。

## ✨ 主な機能

* **🗣️ 音声による温度入力**
* 10秒ごとのビープ音に合わせて温度（例: 「150.5」）を発声するだけで自動記録。
* 音声入力が漏れた場合でも、次回の入力時に自動で線形補間（間をいい感じに繋ぐ）します。


* **🔘 ワンボタン・ステートマシンUI**
* 操作は画面左下のボタン1つのみ。進行状況に合わせて `START` → `CRACK` (1ハゼ) → `STOP` → `SAVE` と役割が自動で変化します。


* **📈 リアルタイム・グラフ描画**
* 温度曲線（Temp）と温度上昇率（RoR）をリアルタイムでプロット。
* RoRは投入直後の温度降下を除き、bottom temperature確定後から表示します。
* 経過時間に合わせてグラフの横幅（タイムスケール）が自動で拡張されます。


* **🎯 目標プロファイル機能**
* 理想の焙煎カーブを事前に入力し、背景にガイドラインとして表示可能。
* 複数の目標プロファイルを保存・切り替えでき、一覧内で名前編集・作成・複製・削除ができます。


* **🔔 Cue設定**
* 10秒ごとのビープ音を設定からオン/オフできます。
* 対応ブラウザでは、ビープ音に加えて振動（Haptic）も利用できます。


* **🔥 ディベロップメント・トラッキング**
* 1ハゼ（CRACK）以降の経過時間と、そこからの温度上昇幅を専用パネルでリアルタイム表示します。


* **⚖️ 生豆・焙煎後重量トラッキング**
* 焙煎前に生豆の投入量（g）を入力し、`STOP` 後に焙煎後重量（g）を入力できます。
* 入力された前後重量から、重量減少率（%）と減少重量（g）を自動計算します。


* **💾 シームレスなCSVエクスポート**
* 焙煎終了後、ボタン1つで全ログデータ（時間、目標温度、実測温度、RoR、bottom情報、1ハゼ情報、焙煎前後重量、重量減少率）をエクセル等で開けるCSV形式で書き出し、即座に次の焙煎（リセット）へ移行できます。


* **📱 スリープ防止機能**
* Wake Lock APIを利用し、焙煎中にスマートフォンの画面が暗くなるのを防ぎます（対応ブラウザのみ）。



## 🚀 使い方

### 1. 準備・起動

本ツールはHTMLファイル1つで動作しますが、**マイク機能（Web Speech API）を利用するため、必ず `https://` で提供されるサーバー（または `localhost`）上で実行してください。**

* **推奨環境:** Google Chrome (Android / PC) または Safari (iOS)
* 初回アクセス時に「マイクの使用許可」を求められるので「許可」を選択してください。

### 2. プロファイルの設定 (任意)

1. 画面右上の `⚙️ Settings` をタップします。
2. `Profile` タブで、目標としたい分数の箇所に温度を入力します（例: `0:00` に `200`、`5:00` に `100` など。未入力の時間は自動で直線状に繋がります）。
3. 複数行の目標値を入力欄に貼り付けると、`150 00:00`、`01:00,80`、`2,105`、CSV行のような形式をまとめて読み込めます。
4. `Apply Profile` を押すとグラフに破線として反映され、ブラウザに記憶されます。
5. プロファイル一覧の `+ Create New` で新規作成、名前欄でリネームできます。左スワイプで削除、長押しで複製できます。

### 3. Cueの設定 (任意)

1. 画面右上の `⚙️ Settings` をタップします。
2. `Cue` タブで、ビープ音と振動（対応ブラウザのみ）を切り替えます。
3. `Test Cue` で現在の設定を確認できます。

### 4. 焙煎のフロー

画面左下のメインアクションボタンをタップして進行します。

1. **投入量入力** : 焙煎前に、生豆の投入量（g）を入力します。
2. **[▶️ START]** : 焙煎開始。10秒ごとに「ピッ」と鳴るので現在の温度を声に出して読み上げます。
3. **[🔥 CRACK]** : 1ハゼが始まった瞬間にタップします。1ハゼのラインが引かれ、ディベロップメントタイムの計測が始まります。
4. **[◼️ STOP]** : 焙煎終了（煎り止め）のタイミングでタップします。タイマーと音声認識が停止し、焙煎後重量の入力が有効になります。
5. **焙煎後重量入力** : 焙煎後の豆重量（g）を入力すると、重量減少率が自動計算されます。
6. **[⬇️ SAVE]** : タップすると今回の焙煎データがCSV形式でダウンロードされ、画面が完全に初期化されます。すぐに次の釜の焙煎（START）に移れます。

## 🛠️ 技術スタック

* **HTML5 / CSS3 / Vanilla JavaScript** (フレームワーク非依存)
* **Chart.js** (v3+) - グラフ描画
* **chartjs-plugin-annotation** - 1ハゼの垂直線描画
* **Web Speech API** - 音声認識
* **Web Audio API** - ビープ音生成
* **Vibration API** - 振動キュー（対応ブラウザのみ）
* **Wake Lock API** - 画面スリープ防止

## 🎨 デザイン方針

UIを追加・変更する場合は、[DESIGN.md](DESIGN.md) の方針に従います。
焙煎中の主画面では roast curve と操作判断に直接効く情報を優先し、内部処理の詳細な数値は原則として CSV や Settings に逃がします。

## ⚠️ 注意事項

* **音声認識の精度について:** 騒音が激しい環境（排気音が極端に大きいなど）では認識精度が落ちる場合があります。その場合はマイク付きイヤホン等の使用を推奨します。
* **振動キューについて:** Vibration APIに対応していないブラウザでは、Haptic設定は利用できません。iOS Safari / iOS Chrome は現状、10秒ごとの自動Haptic Cueには対応していません。
* **iOS (Safari) での制限:** iOSの仕様により、Wake Lock APIが完全に動作しない（画面が暗くなる）場合があります。実運用の際は、iPhone本体の設定で「自動ロック」を一時的にオフにすることをお勧めします。
* **データ保存について:** CSVへの書き出し前にページを再読み込み（リロード）したり閉じたりすると、その回の焙煎データは消失します。必ず `SAVE` ボタンで書き出してください。
